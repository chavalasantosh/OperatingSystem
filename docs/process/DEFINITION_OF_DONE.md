# Definition of Done

A work item is done only when all applicable conditions are met:

- acceptance criteria pass;
- code is formatted and lint-clean;
- unit and integration tests pass;
- emulator boot remains healthy;
- failure paths are handled intentionally;
- unsafe invariants are documented;
- architecture and security impacts are reviewed;
- public interfaces have documentation;
- relevant requirements and ADRs are updated;
- no critical or high-severity known defect is hidden;
- logs contain no unexplained warning or fault;
- recovery and rollback effects are understood;
- a concise demonstration or evidence artifact exists.
